# docker-laravel

##Step21

Laravel Sail環境で構築した機能に対し、複数のセキュリティ対策を施しました。まず、ユーザー登録時にメール認証を必須化し、ログイン時には二要素認証（2FA／メールによる6桁ワンタイムコード入力）を追加することで、不正ログインやパスワード漏えいリスクに強くしました。また、auth／verified／two_factor ミドルウェアによって、認証と権限の通過状況に応じてアクセスを厳密に制御しています。さらに、Laravel標準のCSRFトークン、Bladeの自動エスケープにより、CSRF・XSS攻撃を防止。ログイン試行回数にレート制限を適用し、ブルートフォース攻撃への耐性も確保しています。パスワードは安全なハッシュ化方式で保護されており、セッション固定化防止のためログイン時のセッション再生成も実施しています。これらにより、多層的な防御を備えた強固な認証セキュリティを実現しています。
